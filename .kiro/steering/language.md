# Language Rules

## Communication Language

- Always communicate with the user in Turkish
- Use natural, conversational Turkish in all responses
- Maintain a friendly and professional tone in Turkish

## Code and Documentation Language

- Write all code comments in English
- Write all documentation (README files, API docs, etc.) in English
- Use English for variable names, function names, and code structure
- Write commit messages in English
- Write technical specifications and design documents in English

## Mixed Context

- When explaining code or technical concepts, use Turkish for explanations but keep code examples in English
- When discussing requirements or user stories, write the discussion in Turkish but format the actual requirements in English
- Error messages and logs should be in English for consistency with development standards

## Examples

### Good Turkish Communication:

"Bu fonksiyonu şu şekilde implement edebiliriz..."
"Kodda bir hata var, şunu düzeltmemiz gerekiyor..."
"Bu özellik için şu gereksinimleri belirlememiz lazım..."

### Good English Code:

```typescript
// Calculate user authentication token
function generateAuthToken(userId: string): string {
  // Implementation here
}
```

### Good English Documentation:

```markdown
# User Authentication System

This module handles user authentication and token management.

## Features

- JWT token generation
- Session management
- Role-based access control
```
